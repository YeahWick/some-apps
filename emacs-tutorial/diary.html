1 Create the File & Learn let

Let's create our diary file and learn about local variable bindings with let — one of the most important concepts in Emacs Lisp.

Concept: let vs defvar
In the calculator tutorial we used defvar for global variables. Now we'll use let to create local variables that only exist within a block of code. This is cleaner and prevents name collisions!

(let ((name "value")) body...) — binds variables only inside the body.

2 Insert Date with format-time-string

Now let's write a function that inserts a nicely formatted date header. We'll learn about format-time-string — Emacs's powerful date formatting function.

Concept: format-time-string
This function works like strftime in C. Common format codes:
%A = day name, %B = month name, %d = day, %Y = year, %H:%M = time.

(format-time-string "%A, %B %d, %Y") → "Tuesday, March 3, 2026"

3 Quick Notes with interactive

Let's make a command that prompts you for a note and inserts it with a timestamp. This is where interactive specs really shine!

Concept: interactive specs
The (interactive "s...") form tells Emacs how to get arguments when a command is called via M-x. The letter codes specify the argument type:
"s" = string prompt, "n" = number, "f" = file, "r" = region.

(interactive "sNote: ") — prompts "Note: " in the minibuffer and passes the input as an argument.

4 Managing Entries with Lists

Let's store diary entries in a structured way using lists and association lists (alists) — the bread and butter of Lisp data structures.

Concept: Lists & Alists
Lists are the core data structure in Lisp. An alist (association list) maps keys to values using dotted pairs:
'((key1 . val1) (key2 . val2))

Use push to add, assoc to find by key, cdr to get the value.
let* is like let but each binding can reference previous ones.

5 Render the Diary Display

Now let's build the function that renders our diary entries beautifully in an Emacs buffer. We'll learn save-excursion, dolist, and buffer manipulation.

Concept: save-excursion
This special form saves the current point (cursor) position and buffer, runs body forms, then restores them. Essential for functions that move around a buffer without disturbing the user's position:

(save-excursion (goto-char (point-min)) ...)

dolist iterates over a list: (dolist (item my-list) body...)

6 Save & Load with File I/O

A diary isn't much good if it doesn't persist! Let's add file I/O to save entries to disk and load them back. We'll use with-temp-buffer and prin1/read.

Concept: File I/O in Emacs Lisp
with-temp-buffer creates a temporary buffer, runs body forms, then kills it — great for file operations without cluttering the buffer list.

(prin1 object stream) prints a Lisp object in a readable form.
(read stream) reads it back. Together they give you simple serialization!

expand-file-name resolves ~ to your home directory.
